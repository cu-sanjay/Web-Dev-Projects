Educational Resources & Theory

Memory Fragmentation

Memory allocation algorithms aim to limit wasted space. Fragmentation is divided into:

  • Internal Fragmentation: Space wasted *inside* an allocated partition. Happens in Fixed Partitioning when a process size is smaller than the partition limit. The leftover space in that partition block cannot be accessed by other jobs.
  • External Fragmentation: Space wasted *outside* partitions. Occurs in Dynamic Partitioning when free memory holes are fragmented into small blocks scattered across addresses. The sum of free holes is large enough to serve a process, but because it is not *contiguous*, allocation fails.

Dynamic Coalescing & compaction

To combat external fragmentation, operating systems deploy memory consolidation techniques:

  • Coalescing: When a process terminates and releases its partition block, the simulator checks if adjacent blocks (left or right) are also free. If so, they are merged into a single contiguous free hole.
  • Compaction: Moves all allocated process blocks to one end of the physical address space, merging all scattered free fragments into one large contiguous free block.

Fit Allocation Policies Comparison

  • First Fit: Fast search (stops at first match). Promotes fragmentation at the beginning of memory.
  • Best Fit: Searches full memory to find the closest match. Saves larger blocks, but leaves tiny, unusable fragments.
  • Worst Fit: Allocates the largest available partition. Reduces small fragments but depletes large partitions needed for heavy processes.
  • Next Fit: Continues searches from the last point of allocation. Distributes allocations more evenly but can fragment the end of memory blocks.
