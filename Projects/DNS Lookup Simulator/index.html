The Nameserver Hierarchy

DNS does not store all records in a single database. Instead, it is distributed globally in a tree structure:

  • Root Server (`.`): The top of the tree. There are 13 logical root server addresses globally. They know the addresses of Top-Level Domain (TLD) servers.
  • TLD Server: Manages specific suffixes like `.com`, `.org`, or country codes like `.uk`. They point to the Authoritative server of the requested domain.
  • Authoritative Server: The final destination. Configured by the domain owner, it houses the actual A, AAAA, MX, or CNAME mappings.

Recursive vs. Iterative Queries

Computers query the DNS hierarchy in two distinct patterns depending on client configurations:

  • Recursive Query: The client requests that the Local Resolver handle the entire lookup. The resolver must recursively query nameservers on the internet and return only the final answer.
  • Iterative Query: Nameservers do not perform lookups themselves. Instead, they reply with a **Referral** (e.g. "I don't know the IP of google.com, but TLD `.com` resides at `192.5.6.30`, ask them"). The resolver must perform each subsequent query step-by-step.

Caching & TTL (Time To Live)

To avoid overloading the network with lookups, resolved records are cached locally:

  • TTL (Time-To-Live): A lifespan count in seconds set by the Authoritative nameserver. It dictates how long a local cache is allowed to keep the record before it becomes stale and must be fetched again.
  • Multi-Level Cache: Checks happen in sequence: Browser Cache → OS Hosts Cache → Local ISP Resolver Cache. Only on a complete cache miss does a query go out to the root nameservers.
