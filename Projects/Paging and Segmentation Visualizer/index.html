Address Translation under Paging

In paging, the CPU generates virtual addresses split into a **Page Number ($p$)** and a **Page Offset ($d$)**:

  • Page Size: Determines offset bit lengths ($d = \log_2(\text{Page Size})$).
  • TLB Lookup: A high-speed hardware cache. If mapping for $p$ is in TLB, it's a **TLB Hit**; Frame $f$ is retrieved immediately. Otherwise (TLB Miss), it queries the Page Table.
  • Page Table Lookup: Translates Page $p$ to Frame $f$. If the valid bit is `0`, it causes a **Page Fault**, interrupting the CPU to load the page from Disk to RAM.
  • Physical Address Calculation: $\text{Physical Address} = (f \times \text{Page Size}) + d$.

Address Translation under Segmentation

Segmentation maps virtual memory into logical, variable-sized units (e.g. Code, Stack, Data) matching program structures:

  • Segment Table: Stores the **Base Address** (where the segment starts in physical memory) and **Limit** (maximum length/size of the segment).
  • Address Checking: CPU addresses specify Segment ID $s$ and Offset $d$. The hardware checks:
    • If $d < \text{Limit}$: Safe. $\text{Physical Address} = \text{Base} + d$.
    • If $d \ge \text{Limit}$: **Segmentation Fault Trap** (access violation error).

TLB Replacement Policies

Because TLB cache capacity is small (e.g., 4 slots), loading a new mapping on a TLB Miss requires evicting an entry once slots are full:

  • FIFO (First-In First-Out): Discards the oldest mapping entry in the cache.
  • LRU (Least Recently Used): Tracks usage and discards the mapping that hasn't been referenced for the longest period.
