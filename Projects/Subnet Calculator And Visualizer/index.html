IPv4 Subnetting & CIDR Foundations

Understanding Subnets

An IPv4 address is a 32-bit number, usually written in dotted-decimal format (four 8-bit octets). Subnetting divides the address space into two sections:

  • Network Prefix: Bits used to identify the specific network domain.
  • Host Identifier: Bits used to address individual interface cards within that network.
  • Subnet Masks: Binary filters where 1s specify network bits and 0s represent host bits.

Classless Routing (CIDR)

Classless Inter-Domain Routing (CIDR) replaced the older classful (Class A, B, C) system. It uses a variable prefix syntax (e.g. /24), allowing networks to be sized exactly to requirements:

  • Address Borrowing: Borrowing host bits and treating them as network bits allows a large block to be divided into multiple smaller subnets.
  • Network Boundaries: The Network ID is the first address in the block (all host bits 0). The Broadcast ID is the final address (all host bits 1). These two addresses cannot be allocated to hosts.

VLSM Optimization (Greedy Heuristic)

Variable-Length Subnet Masking (VLSM) allows using different masks for different subnets. It optimizes address utilization:

  • Greedy Allocation: To prevent overlaps, subnets must be allocated descending by size (largest host request first).
  • Alignment Rule: A subnet block of size $S$ (rounded up to the nearest power of 2, including Network & Broadcast overhead) must start at a network address that is a multiple of $S$. Failing to follow this boundary rule creates routing table overlaps.
