The 5 Pipelined Stages

Pipelining breaks the execution of processor instructions into sequential, overlapping steps:

  • IF (Instruction Fetch): Retrieves the instruction opcode from memory at address Program Counter (PC), then updates PC + 4.
  • ID (Instruction Decode): Reads source registers ($Rs$, $Rt$) from the Register File and decodes control bits.
  • EX (Execute): ALU performs computations: add, subtract, logic gates, or compute effective addresses.
  • MEM (Memory Access): Load Word (`LW`) reads data from RAM; Store Word (`SW`) writes data to RAM.
  • WB (Write Back): Writes ALU results or loaded memory words back to the target register ($Rd$ or $Rt$).

Data Hazards & Forwarding

A **Data Hazard** occurs when an instruction reads a register whose value is computed by a preceding instruction that is still in the pipeline.

  • RAW (Read After Write): Instruction I2 reads $R_x$ before I1 has written to it in stage WB.
  • Stalling: Inserting **bubbles** (NOPs) in stage ID/EX to delay I2 until I1 completes WB.
  • Forwarding (Bypass): Optimizes performance by passing ALU inputs directly from EX/MEM or MEM/WB pipeline registers to EX stage inputs, eliminating stalls for standard arithmetic dependencies.
  • Load-Use Stall: When `LW` loads $R_1$, followed immediately by an instruction reading $R_1$, the data is not available until MEM stage. A **1-cycle stall** is mandatory even with forwarding.

Control Hazards & Flushes

A **Control Hazard** arises from conditional branch instructions (`BEQ`). The target address or branching choice is not resolved until the ID/EX stages:

  • Predict Not Taken: The pipeline continues fetching sequential instructions ($PC+4$). If the branch is taken, the incorrectly fetched instructions are **flushed** (overwritten with NOPs/bubbles) in the IF/ID stages.
  • Stalling: Stalls fetching any instruction in IF stage until the branch outcome resolves in the ID stage.
