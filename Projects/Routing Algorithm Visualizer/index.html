Comparative Routing Theory & Concepts

Dijkstra's Link-State (OSPF)

Dijkstra's Shortest Path First (SPF) is the core algorithm used in Link-State protocols like OSPF:

  • Global Knowledge: Every node must know the complete graph topology. Nodes flood Link-State Advertisements (LSAs) containing neighbor details.
  • SPF Tree Construction: Using Dijkstra's algorithm, each router computes a shortest-path tree with itself as the root, determining next-hop gateways.
  • Complexity: $O(V^2)$ or $O(E + V \log V)$ with heaps. Fast convergence and loop-free paths, but memory-intensive.

Bellman-Ford Distance-Vector (RIP)

Bellman-Ford computes paths using distance vectors, forming the basis of RIP:

  • Local Exchange: Routers do not know the full topology. They periodically share their local routing tables with direct neighbors only ("routing by rumor").
  • Distance Vector Formula: Path costs update dynamically: $D_x(y) = \min_v \{ c(x,v) + D_v(y) \}$ where $v$ is a direct neighbor.
  • Convergence Loop Risk: Slow convergence. Prone to routing loops and the **Count-to-Infinity** problem. Resolved using **Split Horizon** and **Poison Reverse** configurations.

Flooding Broadcasting

Flooding is a simple routing protocol where every incoming packet is duplicated and sent on all outgoing links:

  • Loop Prevention: To avoid infinite loops, packets carry unique source sequence identifiers and decrementing **TTL (Time-To-Live)** limits. Routers discard duplicates they have already processed.
  • High Overhead: Guaranteed to find the shortest path (minimum latency) because all paths are checked simultaneously, but creates massive traffic congestion and packet storms.
