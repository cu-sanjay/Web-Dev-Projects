Network Architecture & Encapsulation Theory

OSI / TCP-IP 5-Layer Encapsulation

When a computer sends data across a network, it moves down the protocol stack. At each layer, header information is wrapped around the payload:

  • L5 Application: Creates raw messaging payload (e.g., HTTP requests or DNS queries).
  • L4 Transport: Standardizes ports. TCP adds sequence fields for reliability; UDP adds simple checksum envelopes.
  • L3 Network: Appends IP headers. Identifies routing targets globally using IP addresses.
  • L2 Data Link: Appends Ethernet II headers. Connects neighboring NIC devices locally using physical MAC addresses.
  • L1 Physical: Converts final frame blocks into binary bit streams (`0`/`1`) carried over electrical lines.

Switching vs Routing Hop Transformations

As a packet moves across nodes in a network, different devices read different headers to make forwarding decisions:

  • Layer 2 Switch: A local connector. It decapsulates the packet to Layer 2, reads the Destination MAC address, and forwards the frame out the correct port without reading IP addresses.
  • Layer 3 Router: A gateway between networks. It strips the L2 frame to read the L3 IP header. It performs a routing table lookup, decrements the **TTL (Time-to-Live)** field by 1, re-encapsulates the IP packet inside a *new* L2 Ethernet frame (with its own MAC as source and the next hop's MAC as destination), and transmits the packet.

Packet Headers Breakdown

Headers contain control metadata required for the packet to reach its destination and be reassembled:

  • Ports (L4): Identifies the specific application process in the OS (e.g., Port 80 for HTTP web traffic, Port 53 for DNS queries).
  • IP Addresses (L3): Identifies logical host addresses across separate networks globally.
  • MAC Addresses (L2): Hardcoded physical hardware addresses identifying local interfaces on the same network link segment.
  • TTL (L3): A counter decremented by every router to prevent infinite packet loops. If TTL hits 0, the router drops the packet.
