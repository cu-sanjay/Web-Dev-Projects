Virtual Memory Architectures & Core Mechanics

Address Translation Math

Virtual Memory divides a process's addresses into uniform blocks called Pages, while Physical Memory (RAM) is split into matching blocks called Frames.

  • An 8-bit Virtual Address yields a 256-byte space.
  • With 32-byte pages, we calculate the bits: $32 = 2^5$, which means **5 bits are used for the Offset**.
  • The remaining $8 - 5 = 3$ bits represent the **Virtual Page Number (VPN)**, allowing for $2^3 = 8$ pages (Page 0 to Page 7).
  • The translation translates VPN → PFN, leaving the Offset unchanged: $\text{Physical Address} = (\text{PFN} \times \text{PageSize}) + \text{Offset}$.

Understanding TLB Mechanics

The **Translation Lookaside Buffer (TLB)** is a small, fast hardware cache inside the CPU's memory management unit (MMU).

  • TLB Hit: The VPN lookup finds an entry in the TLB. Translation happens in less than a cycle, bypassing the page table lookup.
  • TLB Miss: The VPN lookup is not cached. The CPU must traverse the multi-step Page Table in RAM, taking tens of cycles. The translation is then loaded into the TLB for future access.
  • If the TLB is full, an existing entry must be evicted using cache replacement rules (LRU, FIFO, Clock).

Page Faults & Eviction Flags

A **Page Fault** occurs when the virtual page accessed has its **Valid bit set to 0**, meaning the page resides on secondary disk storage, not in RAM.

  • Page Fetch: The OS pauses execution, reads the page from the Swap Disk, and loads it into an empty RAM Frame.
  • Page Eviction: If all RAM frames are occupied, a victim page is evicted using the selected replacement policy.
  • Dirty Write-back: If the evicted page has its **Dirty bit set to 1** (meaning it was modified in RAM), it must be written back to the Swap Disk. If it is clean (Dirty = 0), the space can be overwritten immediately.
