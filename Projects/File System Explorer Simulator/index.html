File System Architecture & Core Mechanics

Directory Structures & Navigation

Operating systems organize storage files into structured recursive trees.

  • Root Node (`/`): The absolute start of a partition's directory structure.
  • Directory Entries: Folders are special files containing lists of mappings matching sub-files to their respective **inodes**.
  • Path Resolution: To access `/home/user/document.txt`, the OS translates each sub-path from left to right. If any step is missing or has incorrect permissions, path resolution fails.

OS Inodes & File Control Blocks (FCB)

An **inode** (index node) is a data structure on disk that describes a file system object (file or directory).

  • Inodes do **NOT** store the filename. Filenames are stored inside directories mapped to inode index pointers.
  • Inodes contain essential metadata: object type (file/folder), file size in bytes, device identifier, access permissions (read/write/execute), and timestamps.
  • In a real partition, inodes also contain data block pointers locating where the raw data blocks are on the physical drive.

Storage Allocation Methods

The Operating System allocates physical disk blocks to files in three major ways:

  • Contiguous Allocation: Each file occupies a set of consecutive blocks on disk. Fast sequential access, but prone to external fragmentation.
  • Linked Allocation: Files are stored as linked lists of blocks scattered across the disk. Solves fragmentation but random access is slow.
  • Indexed Allocation: Consolidates all block pointers into a single block called the **Index Block** (linked to the Inode). Perfect for fast random access and indexing, but wastes block space for tiny files.
